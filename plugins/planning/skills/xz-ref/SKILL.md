# XZ Ref - 加载计划作为参考上下文

参数: `$ARGUMENTS`

### 参数校验

如果 `$ARGUMENTS` 为空，**立即停止**，提示：

> 缺少版本号。用法: `/xz-ref N` 或 `/xz-ref N1,N2,N3`
> 示例: `/xz-ref 1` 或 `/xz-ref 1,2,3`

支持单个版本或逗号分隔的多个版本号，如 `1` 或 `1,2,3`。

---

## 辅助脚本

插件 `bin/` 目录下的 `xz-tools.py`（插件启用时自动加入 PATH）

---

## 执行流程

### 第一步：解析版本号列表

将 `$ARGUMENTS` 按逗号分隔，得到版本号列表。如 `1,2,3` → `[1, 2, 3]`，`1` → `[1]`。

### 第二步：逐个查找 PLAN 文件

对每个版本号 N，执行：

```bash
xz-tools.py parse N
```

查找范围包括：
- `.xz_planning/phases/N.xxx/N-PLAN.md`（活跃版本）
- `.xz_planning/archive/N.xxx/N-PLAN.md`（已归档版本）

如果某个版本不存在，提示跳过但继续处理其他版本。

### 第三步：读取并加载

读取所有找到的 N-PLAN.md 完整内容，加载到当前对话上下文中。

同时读取同目录下的 N-DISCUSS.md 和 N-UAT.md（如果存在）。

### 第四步：输出加载摘要

```
已加载参考:
  [1] 用户注册登录    5 条 todo (5/5 完成) + UAT
  [2] 历史开关        3 条 todo (1/3 完成)
  [3] 未找到，已跳过

当前上下文已包含以上计划内容，可以直接提问或引用。
```

---

## 用途

加载后 AI 即拥有这些计划的完整上下文，你可以：

- 直接提问："版本 1 的注册接口是怎么设计的？"
- 跨版本对比："版本 1 和版本 2 有哪些文件重叠？"
- 辅助新计划："参考版本 1 的风格，帮我规划版本 4"
- 排查问题："版本 2 的 #3 改动会不会影响版本 1 的逻辑？"

## 下一步建议

加载完成后提示：

```
已加载版本上下文，可以直接提问或引用。

相关命令:
  /xz-exec N              ← 执行 todolist
  /xz-update-plan N ...   ← 修改计划
  /xz-plan N ...           ← 创建新版本计划
```

## 关键规则

1. **只读不写** — 不修改任何文件
2. **活跃 + 归档都查** — phases 和 archive 目录都找
3. **找不到就跳过** — 不中断，继续加载其他版本
4. **附带 DISCUSS + UAT** — 如果有 N-DISCUSS.md 和 N-UAT.md 一并加载
